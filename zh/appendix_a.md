## A.1 模型选择速查

选择合适的大模型是 Agent 开发的第一步。就像选汽车发动机——马力不是越大越好，适合你的使用场景才最重要。

| 类型 | 代表产品 | 特点 | 适用场景 |
|-|-|-|-|
| 通用大模型 | GPT-4 / Claude / Gemini | 能力全面，推理强，综合表现最好 | 通用 Agent、复杂任务、多步骤推理 |
| 代码模型 | Codex / Claude Code / Cursor | 代码理解与生成能力突出，熟悉编程规范 | 编程 Agent、代码审查、自动化测试 |
| 开源模型 | Llama / Qwen / Mistral | 可私有化部署，数据安全，成本可控 | 企业内网、敏感数据场景、定制化需求 |
| 多模态模型 | GPT-4V / Claude 3 Opus / Gemini Pro | 图文理解，能看懂图片、图表、设计稿 | 设计 Agent、视觉识别、UI 还原、截图分析 |
| 轻量模型 | GPT-4o mini / Claude Haiku / Qwen-Turbo | 速度快、成本低，适合简单任务 | 分类、摘要、简单路由、批量处理 |

💡 选型建议

不要盲目追求"最强模型"。最佳实践是**分层使用**：简单任务用轻量模型省钱提速，复杂任务用强模型保证质量，关键节点再用最强模型做 Review。一个好的 Agent 系统，往往是"多模型协同"的结果。

## A.2 Agent 开发框架

如果说模型是发动机，框架就是整车的底盘和装配线。选对框架，能帮你少走很多弯路。

| 框架 | 语言 | 特点 | 适用场景 |
|-|-|-|-|
| **LangChain / LangGraph** | Python / JS | 生态最丰富，社区最大，文档最全 | 快速原型、学习入门、通用场景 |
| **AutoGPT** | Python | 自主性强，目标驱动，自动拆解任务 | 探索实验、研究用途、开放式任务 |
| **CrewAI** | Python | 多 Agent 协作，角色分工清晰 | 团队 Agent、复杂工作流、多角色协作 |
| **OpenAI Agents SDK** | Python | OpenAI 原生，深度集成 GPT 能力 | OpenAI 生态、简单 Agent、快速上手 |
| **Dify** | 可视化 / 低代码 | 拖拽式操作，非技术人员也能用 | 非技术人员、业务人员、快速验证想法 |
| **LlamaIndex** | Python | RAG 能力强，数据连接丰富 | 知识库 Agent、文档问答、企业搜索 |
| **AutoGen** | Python | 微软出品，多 Agent 对话框架 | 研究导向、多智能体对话、复杂协作 |

🔬 内行建议

新手入门推荐 **LangChain**——教程最多，踩坑了也容易找到答案。如果是做企业级应用，建议重点关注 **LangGraph**，它的状态机模型更适合构建可靠的 Agent 流程。非技术同学可以从 **Dify** 开始，零代码也能搭出像模像样的 Agent。

## A.3 关键工具链

有了模型和框架，你还需要一整套工具链来支撑 Agent 的开发、运行和监控。就像汽车需要加油站、维修厂、导航系统一样。

🗄️

向量数据库

Pinecone / Chroma / Weaviate / Milvus

存储和检索向量嵌入，是 RAG 系统的核心基础设施。轻量场景用 Chroma，企业级用 Pinecone 或 Milvus。

🔌

MCP 服务

官方 MCP Server / 社区 MCP

Model Context Protocol——Agent 接入工具的标准协议。一次接入，到处可用，正在成为行业标准。

💻

代码 Agent

Claude Code / Codex / Cursor

AI 编程助手，能理解项目结构、写代码、跑测试、修 Bug。开发者的"副驾驶"，日常开发效率神器。

📊

可观测平台

LangSmith / Langfuse / Helicone

Agent 的"行车记录仪"。记录每一次调用、追踪成本、调试错误、评估效果。生产环境必备。

🛡️

沙箱环境

Docker / E2B / Firecracker

Agent 执行代码的安全隔离环境。防止 Agent 乱跑命令、误删文件、泄露数据。安全底线。

📝

Prompt 管理

PromptLayer / Humanloop / Galileo

集中管理 Prompt 模板、做 A/B 测试、追踪 Prompt 效果。当你有几十上百个 Prompt 时就知道它的好。

⚠️ 注意

工具链不是越多越好。初期建议从**模型 + 框架 + 一个可观测工具**开始。等 Agent 跑起来了，再根据实际痛点逐步补齐其他工具。过早引入复杂工具链，反而会增加学习成本和维护负担。

## A.4 常用 Prompt 模板

Prompt 是 Agent 的"操作手册"。好的 Prompt 模板能让 Agent 的表现提升一个档次。以下是实战中最常用的五个模板，直接套用即可。

1. 角色设定模板 基础必用

你是一位[资深前端工程师]，拥有[8年]相关经验。 你的专长是[React + TypeScript + 性能优化]。 工作原则： - 代码必须简洁、可读、可维护 - 优先考虑性能和用户体验 - 每个决定都要有理由 请以专业、严谨的态度回答问题，必要时给出代码示例。

2. 任务拆解模板 复杂任务

任务目标：[完成一个用户登录功能] 请按照以下步骤处理： 1. 先理解需求，列出关键要点 2. 将任务拆解为具体的子任务（不超过 5 个） 3. 按优先级排序，说明每个子任务的依赖关系 4. 逐一执行子任务 5. 完成后做整体检查 每完成一个子任务，汇报进度和结果。遇到不确定的地方，停下来问我。

3. Reviewer 审查模板 质量保障

你是一位严格的代码审查员。请审查以下代码，从这几个维度评估： 🔴 严重问题（必须修改）： - 逻辑错误或安全漏洞 - 明显的性能问题 - 不符合编码规范 🟡 建议改进（可选修改）： - 代码可读性可以提升的地方 - 更优雅的实现方式 - 边界条件考虑不周 🟢 做得好的地方： - 值得肯定的实现 请给出具体的修改建议和代码示例。不要笼统，要具体到行。

4. 结构化输出模板 格式规范

请分析以下内容，并以 JSON 格式输出结果。 输出格式要求： \`\`\`json { "summary": "一句话总结", "key_points": ["要点1", "要点2", "要点3"], "sentiment": "positive/neutral/negative", "action_items": [ { "task": "具体任务", "priority": "high/medium/low", "deadline": "建议完成时间" } ] } \`\`\` 要求： - 严格按照 JSON 格式输出，不要有额外文字 - key_points 不超过 5 条 - action_items 不超过 3 条

5. 风险评估模板 决策辅助

请对以下方案/决策进行风险评估： 方案描述：[使用 LangChain 构建企业级 Agent 系统] 请从以下维度分析： 1. 技术风险：技术选型是否合理？有没有坑？ 2. 成本风险：开发和运维成本大概多少？是否可控？ 3. 时间风险：预计多长时间能上线？可能延期的原因？ 4. 安全风险：有没有数据安全或权限问题？ 5. 替代方案：如果这个方案不行，还有什么备选？ 最后给出总体评估：✅ 推荐 / ⚠️ 谨慎推荐 / ❌ 不推荐，并说明理由。

✅ 使用技巧

Prompt 模板不是一成不变的。建议你在使用过程中**持续迭代优化**——记录哪些 Prompt 效果好、哪些不行，慢慢打磨出属于自己的"Prompt 军火库"。另外，记得把常用 Prompt 放到 `AGENTS.md` 或专门的配置文件里，方便随时调用。
