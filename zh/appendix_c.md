共收录 **33** 个关键术语

A

- Agent 智能体
- 能自主理解任务、制定计划、使用工具并完成目标的 AI 系统。和传统 Chatbot 的区别在于：Chatbot 只是"回答问题"，Agent 是"完成任务"——它真的会动手做事。 就像一辆能自己开的智能汽车，你告诉它目的地，它自己规划路线、控制方向、避开障碍，把你送到地方。
- Agent Mesh 智能体网状网络
- 多个 Agent 组成的去中心化协作网络。每个 Agent 有自己的专长和角色，它们之间可以互相通信、分工协作，共同完成复杂任务。不是上下级的层级关系，而是平等的网状连接。 就像一个车队，有负责开路的、有负责运货的、有负责维修的，它们通过无线电互相通信，自动组队完成运输任务。
- Agent OS 智能体操作系统
- 运行和管理 Agent 的基础软件平台。就像 Windows 是电脑的操作系统一样，Agent OS 负责管理 Agent 的生命周期、资源分配、任务调度、安全隔离等。 就像汽车的整车控制系统——管着发动机、刹车、导航、娱乐系统，让所有部件协同工作。

B

- Builder / Reviewer 构建者 / 审查者模式
- 一种经典的双 Agent 协作模式：Builder 负责生成内容（写代码、写文档），Reviewer 负责审查和反馈。两者交替工作，通过"生成-审查-修改"的循环不断提升质量。 就像建筑工地上，工人砌墙（Builder），监理检查质量（Reviewer），不合格的拆掉重砌，合格的继续往上盖。

C

- Chain of Thought (CoT) 思维链
- 一种让大模型"一步步想"的 Prompt 技术。不是直接要答案，而是让模型把推理过程说出来——先想什么、再想什么、最后得出什么结论。这样模型的准确率会大幅提升。 就像开车时不能只看终点，要一步一步来：先看红绿灯、再看左右来车、然后踩油门、接着变道……分步思考就不容易出错。
- Context 上下文
- 大模型在做决策时能"看到"的所有信息。包括当前对话、历史消息、系统提示、检索到的文档等等。上下文的质量和数量直接影响 AI 的表现。 就像汽车前挡风玻璃外的视野——你能看到多少路、多少车、多少交通标志，决定了你开车的安全性和准确性。
- Context Engineering 上下文工程
- 研究如何给 AI 提供最合适的上下文信息的工程学科。包括怎么组织信息、怎么筛选相关内容、怎么排列优先级，目标是用有限的上下文窗口达到最好的效果。 就像给司机设计仪表盘和挡风玻璃——不是信息越多越好，而是该看到的都能看到，无关的别来干扰。
- Context Window 上下文窗口
- 大模型一次能处理的最大 Token 数量。就像一个"信息背包"，装得下多少东西是有限的。窗口越大，AI 能"看到"的信息越多，但成本也越高。 就像汽车的挡风玻璃大小——玻璃越大，视野越开阔，但车也越贵、风阻也越大。

E

- Embedding 嵌入 / 向量
- 把文字、图片等内容转换成一串数字（向量）的过程。这串数字代表了内容的"语义特征"——意思相近的内容，它们的向量在数学空间里也离得近。这是 RAG 技术的基础。 就像把地址转换成经纬度坐标——两个地方意思上近不近，看坐标距离就知道了。向量就是语义的"经纬度"。

F

- Function Calling 函数调用
- 大模型的一种能力：它不只是输出文字，还能"决定"调用一个外部函数/工具，并生成调用参数。这是 Agent 能"动手做事"的关键技术——让 AI 从"只会说"变成"也会做"。 就像司机不只是嘴上说"我要转弯"，而是真的去打方向盘、踩刹车——Function Calling 就是 AI 的"手和脚"。

H

- Harness 驾驭系统
- 控制和约束 Agent 行为的工程系统。包括权限控制、行为规则、验证机制、人工接管点、失败恢复等等。核心目标是让 Agent 安全、可靠、可控地工作。 就像汽车的刹车、方向盘、安全带、安全气囊、交通法规——没有这些，车跑得越快越危险。
- Harness Engineering 驾驭工程
- 研究如何设计和实现 Harness 系统的工程学科。核心问题是：怎么让 Agent 既能做事，又不闯祸？怎么在自主性和可控性之间找到平衡？ 就像汽车安全工程——研究怎么设计刹车系统、怎么做碰撞测试、怎么制定安全标准。
- Human in the Loop 人在环中
- 一种 Agent 工作模式：关键决策点需要人类确认或干预。不是完全放手让 AI 自己干，而是人在关键节点把关——AI 做执行，人做决策。 就像自动驾驶的 L2/L3 级别——车可以自己开，但遇到复杂路况还是要人接管，关键时刻人说了算。

I

- In-Context Learning 上下文学习
- 大模型的一种神奇能力：不需要重新训练，只要在对话里给几个例子，它就能学会新任务。所有学习都发生在"上下文窗口"里，任务结束就忘了。 就像你坐进一辆出租车，跟司机说"今天走这条路，因为那边在修路"——司机一听就明白了，不需要去驾校重新学。

L

- Loop 循环
- Agent 自主运行的基本模式：感知→思考→行动→观察→再思考→再行动……不断循环，直到任务完成。有了 Loop，Agent 才能"自己跑起来"，而不是你推一下动一下。 就像汽车的发动机循环——吸气、压缩、做功、排气，不断重复，车才能持续前进。Agent 的 Loop 就是它的"发动机"。
- Loop Engineering 循环工程
- 研究如何设计 Agent 循环的工程学科。核心问题是：循环里应该有哪些步骤？什么时候继续循环、什么时候停下来？怎么避免死循环？怎么优化循环效率？ 就像汽车的动力系统工程——研究发动机怎么设计、变速箱怎么匹配、巡航控制怎么实现，让车跑得又快又省油。
- LLM (Large Language Model) 大语言模型
- 用海量文本数据训练出来的大型神经网络模型。它能理解和生成人类语言，是当前 AI 革命的核心技术。GPT、Claude、Llama 都是 LLM。 就像汽车的发动机——是整个系统的动力核心，所有其他部件都是围绕它工作的。

M

- MCP (Model Context Protocol) 模型上下文协议
- Anthropic 提出的一种开放协议，用来标准化 AI 模型和外部工具/数据源之间的连接方式。有了 MCP，工具一次接入，就能在任何支持 MCP 的 AI 客户端里使用。 就像汽车的 OBD 接口（车载诊断接口）——标准接口，任何兼容的设备都能插上去读取车辆数据、控制车辆功能。
- Memory 记忆
- Agent 存储和回忆信息的能力。通常分为短期记忆（当前对话的上下文）和长期记忆（存储在外部数据库里的历史知识）。记忆是 Agent 能"学习"和"积累经验"的基础。 就像司机的记忆——短期记忆是当前路况（记几秒几分钟），长期记忆是你走过的路、学过的交规（记几个月几年）。

O

- Observability 可观测性
- 能实时看到 Agent 在做什么、为什么这么做、花了多少钱、出了什么错的能力。可观测性是调试和运维 Agent 系统的基础——你看不见它，就管不住它。 就像汽车的仪表盘和行车记录仪——仪表盘告诉你现在车速、转速、油量，行车记录仪记录你走过的路，出了事故可以回看。

P

- Prompt 提示词
- 你给 AI 的输入文字。可以是一个问题、一个指令、一段描述，或者一整套规则。Prompt 的好坏直接影响 AI 输出的质量。 就像你跟司机说的话——"去机场"是简单的 prompt，"走高架，避开拥堵，我赶时间"是更详细的 prompt。说得越清楚，司机开得越准。
- Prompt Engineering 提示词工程
- 研究怎么写好 Prompt 的技术和方法。通过精心设计输入，让 AI 输出更好的结果。是 Agent 技术演化的第一代范式。 就像学车时研究怎么跟教练说话——怎么说才能让教练准确理解你的意思，教得更好。

R

- RAG (Retrieval-Augmented Generation) 检索增强生成
- 一种技术：AI 回答问题之前，先去外部知识库搜索相关信息，把搜索结果放进上下文，再基于这些信息生成回答。这样 AI 就能引用最新的、私有的知识，而不是只靠训练时记住的东西。 就像司机开车时用导航——不是靠脑子里记的路，而是实时从地图数据库里检索最新路况，然后规划路线。
- ReAct 推理与行动
- Reasoning + Acting 的缩写。一种 Agent 工作模式：先思考（推理），再行动，然后观察结果，再思考下一步……推理和行动交替进行。是很多 Agent 框架的基础模式。 就像开车时：先观察路况（思考），然后打方向盘踩油门（行动），然后看车走到哪了（观察），再调整方向（再思考再行动）。

S

- Sandbox 沙箱
- 一个隔离的安全执行环境。Agent 在沙箱里运行代码、操作文件，即使搞砸了也不会影响到外面的真实系统。是 Agent 安全的重要基础设施。 就像驾校的练车场——有围墙、有专门的场地，新手司机在里面练车，撞了也不会伤到路人、不会破坏公共设施。
- Skill 技能模块
- Agent 可以调用的、封装好的专用能力模块。比如"写邮件"技能、"做表格"技能、"画图表"技能。技能是可复用的，不同的 Agent 可以共享同一套技能库。 就像汽车的各种功能模块——空调系统、音响系统、导航系统、雨刷系统……想用哪个开哪个，都是独立封装的。
- Sub-agent 子代理
- 被主 Agent 派出去执行特定子任务的小 Agent。就像老板派员工去干活一样，主 Agent 负责统筹，子 Agent 负责具体执行某件事。做完了把结果汇报给主 Agent。 就像车队里的小组车——队长车（主 Agent）负责整体规划，需要的时候派出小组车（子 Agent）去探路、送货、查路况。

T

- Token 词元
- 大模型处理文本的基本单位。一个 Token 可以是一个字、半个词、或者一个完整的词。英文里 1 个 Token 大约是 0.75 个单词，中文里 1 个 Token 大约是 1-2 个汉字。Token 的数量直接关系到成本和上下文窗口大小。 就像汽车的"公里数"——你开了多少公里，就要付多少油钱。Token 就是 AI 的"公里数"，用了多少 Token，就要付多少钱。

V

- Verification 验证
- 检查 Agent 输出结果是否正确、是否符合要求的过程。可以是自动验证（跑测试、检查格式），也可以是人工验证（人来审核）。验证是 Harness 系统的核心环节。 就像汽车年检——检查刹车灵不灵、灯光亮不亮、排放合不合格。不验证，你就不知道这车能不能安全上路。
- Vector Database 向量数据库
- 专门存储和检索向量（Embedding）的数据库。它的核心能力是"语义搜索"——输入一句话，找出意思最相近的内容。是 RAG 系统的核心组件。 就像地图导航的数据库——存的不是纯文本地址，而是经纬度坐标，能快速找到"离我最近的加油站"。

W

- World Model 世界模型
- AI 对真实世界运作方式的理解和建模。一个好的世界模型能让 Agent 预测"如果我这么做，会发生什么"，从而提前规划、避免错误。是高级 Agent 的重要能力。 就像老司机的经验——开过很多路，知道什么路况会出什么事，能提前预判风险。世界模型就是 AI 的"驾驶经验"。
- Workflow 工作流
- 一系列按固定顺序执行的步骤。每个步骤做什么、输入输出是什么、下一步怎么走，都是预先定义好的。优点是可靠可控，缺点是不够灵活。 就像公交路线——站点和路线都是固定的，准时准点，但只能走固定线路，遇到突发情况没法灵活调整。
- Worktree 隔离工作区
- Git 的一个功能，允许你在同一个仓库里同时打开多个分支，每个分支有独立的工作目录。在 Agent 场景下，Worktree 常用来给不同的子 Agent 提供隔离的工作环境，避免互相干扰。 就像车队里的每辆车有自己的车道和任务——各干各的，互不干扰，最后在终点汇合。
